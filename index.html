日记

发表于 2021-06-24

其实严格来说不算日记，算是杂烩吧，写周报隔得时间比较长，容易断更，还是这种随手以及的形式更好

2021/6/24

心血来潮开了个新坑，周报已经删掉了。经过这么多年的网上冲浪后，我发现上帝根本没必要用语言来阻止人类修建巴别塔，即使人类只剩三个人也无法停止诽谤，剩下两个人也无法停止争论，所以毁灭吧赶紧的，我认为我不属于任何人事物，没有任何权力和责任，我只能是我，甚至我都不能是我，因为语言不足以形容我，我身上没有也不可能有任何标签，甚至没有标签本身都没有

最终引导我写下这段话是因为我喜爱的视频博主回形针paper clip卷入了一连串事件，整个事件的时间跨度比较长，说实话我无法为这一整件事定性是好是坏，我也没有这个能力，我不知道它到底有没有错，不关心肉蛋奶，雨林还是台湾什么的，我不知道，也不关心，我只想好好看科普，或者是好好看我想看的内容，它的正确，错误，误导，我都无从判断，也许以后我也会对它产生强烈的厌恶，但是谁在乎呢？至少现在的我不在乎，作为观众，我不是带着语言，国家，抑或是任何任何的标签去看一个视频，即使外界评论，那些我常浏览的地方硬要套个标签给我的话，也只能是一个回形针的观众

我不希望我被任何一种语言思想描述，我不是一个生物，不是一个人，不是一个中国人，我是我

小说

发表于 2021-02-12 更新于 2021-04-29

恋

不知道该怎么写，先把想写的东西说出来吧。我想写物化的女性和脱离女性的女性，双开吧，中世纪和现代
当前印象较深的是本子里的剧情，一位女性虔诚的跪着，出于爱慕求男主大发慈悲，不是处于恐惧或一些本子常见套路莫名戳中了我的某个东西，就此这个切片开始发散吧

生产力低下，魔法架构，物化，权利，多文化,多宗教,单一神创论,历史

我睡到自然醒,阳光照进床边,在床上翻弄了一会儿,我呼唤门口的仆从,门把随即转动,几位仆从稍作礼节性问候后询问我是否要更换衣物,在我简单答复后,她们便开始工作

与其凭空想象,不如创造历史吧

原初创世,神(即我,严格来说)是半神,神诞生于世界出现很久之后,基本沿袭地球,世界既是唯物的也是唯心的,划分到一众架空世界的魔法体系里应该算是低魔或者无魔力,姑且在这提一嘴
现存的神,主要任务是给各种事物发任务卡的,应该不会机械降神
魔法架构主要依靠世界层的叠加,描写出来的世界并不能直观的展现世界观,理论上每个事物都是神,至少是神的一部分,即使从比例上看是很小一个部分,但神的取值是无限大,所以理论上每个事物都是神,并不会有明显的奇迹发生,例如火球术冰锥术之类的古典法术,也不会有改变概率世界空间的奇淫巧计,文章里出现的与魔法相关的大多是神职人员或世俗人员的祈祷,而且并不会出现什么神奇的效果
虽然说是单一神创世,但还是有多宗教,这主要和神会在多地不同时间段发任务卡有关

准备工作

  • GitHub账号

本文所运行的环境

  • Windows 10

创建仓库

在GitHub的右上角,使用下拉菜单选择 New repository(新建仓库)

使用 Owner(所有者)下拉菜单选择你想要拥有仓库的帐户

输入仓库的名称和说明(可选),如果您创建的是用户或组织站点,仓库名称必须为<user>.github.io 或<organization>.github.io

单击 Create repository(创建仓库)

配置SSH key

检查现有 SSH 密钥

  1. 打开 Git Bash
  2. 输入 ls -al ~/.ssh 以查看是否存在现有 SSH 密钥:
  1. 检查目录列表以查看是否已经有 SSH 公钥.默认情况下,公钥的文件名是以下之一
  • id_rsa.pub
  • id_ecdsa.pub
  • id_ed25519.pub

如果您没有现有的公钥和私钥对,或者不想使用任何可用于连接到 GitHub 的密钥对,则生成新的 SSH 密钥

生成新 SSH 密钥

  1. 打开 Git Bash
  2. 粘贴下面的文本(替换为您的 GitHub 电子邮件地址)

  3. 注：如果您使用的是不支持 Ed25519 算法的旧系统,请使用以下命令：

这将创建以所提供的电子邮件地址为标签的新 SSH 密钥

3.提示您“Enter a file in which to save the key(输入要保存密钥的文件)”时,按 Enter 键 这将接受默认文件位置

  1. 在提示时输入安全密码

将 SSH 密钥添加到 GitHub 帐户

  1. 从C:\Users\YourUserName.ssh打开.pub拓展名的文件将 SSH 公钥复制到剪贴板

  2. 在GitHub的右上角,单击您的个人资料照片,然后单击 Settings(设置)

  3. 在用户设置侧边栏中,单击 SSH and GPG keys(SSH 和 GPG 密钥)

  4. 单击 New SSH key(新 SSH 密钥)或 Add SSH key(添加 SSH 密钥)

  5. 在”Title”(标题)字段中,为新密钥添加描述性标签 例如,如果您使用的是个人 Mac,此密钥名称可能是 “Personal MacBook Air”

  6. 将密钥粘贴到 “Key”(密钥)字段

  7. 单击 Add SSH key(添加 SSH 密钥)

  8. 如有提示,请确认您的 GitHub 密码
